# Инструкция пользователя
## Система управления производством — ТОО KUS & ТОО SSAP

---

## 1. Что это

Веб-приложение для ежедневного ввода и анализа производственных показателей двух предприятий:

- **ТОО KUS** — производство электроэнергии (кВч) и очистка воды (м³)
- **ТОО SSAP** — производство серной кислоты (т)

Все данные хранятся локально в браузере (`localStorage`) — установка и интернет-соединение
(кроме первой загрузки) не требуются.

---

## 2. Как запустить

**Способ 1 — локально (самый простой):**
1. Распакуйте архив в любую папку.
2. Дважды кликните по файлу `production_app.html`.
3. Приложение откроется в браузере по умолчанию.

> При первой загрузке нужен интернет — подгружаются библиотеки React и Recharts с CDN.
> Далее браузер кэширует их, и приложение работает офлайн.

**Способ 2 — по ссылке (онлайн):**
Откройте опубликованную версию: https://ermekkasymov.github.io/production-management-system/

**Способ 3 — на внутреннем сервере:**
Загрузите `production_app.html` (или весь набор файлов) на веб-сервер и откройте по адресу
`http://адрес-сервера/production_app.html`.

**Поддерживаемые браузеры:** Chrome 90+, Firefox 88+, Safari 14+, Edge 90+ (включая мобильные).

---

## 3. Как пользоваться

### 3.1 Ввод данных (вкладка «📈 Дашборд»)

1. Выберите предприятие — кнопки **ТОО KUS** или **ТОО SSAP**.
2. Дата по умолчанию — сегодня (можно выбрать прошлую дату).
3. Заполните показатели. Поля со звёздочкой **\*** обязательны:
   - **KUS:** выработка факт/план (кВч), очищенная вода (м³), качество воды (опц.).
   - **SSAP:** серная кислота факт/план (т), концентрация % (опц., 0–100), качество (опц.).
4. Нажмите **«💾 Сохранить данные»**.
   - Появится зелёное уведомление об успехе, форма очистится.
   - Если обязательное поле пустое — появится сообщение об ошибке, данные не сохранятся.
5. Ниже обновятся карточки «Последние показатели» и «История записей» (последние 10).

**Цветовая индикация % выполнения плана:**
- 🟢 Зелёный — выполнение ≥ 95% (норма)
- 🟠 Оранжевый — выполнение < 95% (требует внимания)

### 3.2 Анализ (вкладка «📋 Общий отчёт и анализ»)

1. Задайте **Дату начала** и **Дату окончания**, нажмите **«Применить фильтр»**
   (или **«Сбросить»** для полного диапазона).
2. Смотрите:
   - **Итоговую статистику** за период (суммы факт/план и % по KUS и SSAP).
   - **График «Выполнение плана»** — % по дням, пунктир = цель 95%.
   - **График «Сравнение производства»** — столбцы факт/план (KUS слева, SSAP справа).
   - **Таблицу** всех записей за период.

---

## 4. Резервное копирование данных

Данные хранятся только в вашем браузере. Чтобы не потерять их при очистке кэша:

- **⬇ Экспорт** (кнопка в шапке) — сохраняет все записи в файл `production_data_ГГГГ-ММ-ДД.json`.
- **⬆ Импорт** — загружает записи из ранее сохранённого JSON-файла.

> Рекомендуется делать экспорт регулярно (например, раз в неделю).

---

## 5. Важные ограничения (версия 1.0)

- Данные хранятся локально и **не синхронизируются** между компьютерами и браузерами.
- Нет аутентификации и ролей — доступ имеет любой, кто открыл приложение.
- Рекомендуемый объём — до ~10 000 записей.
